# ערוצי תמיכה (Support Channels)

## 1. סקירה

הלקוח יכול לפנות לתמיכה ב-4 ערוצים. כל ערוץ מתאים לסוג שונה של פניה ולחומרת תקלה שונה.

| ערוץ | זמינות | חומרה רלוונטית | זמן תגובה ראשון |
|---|---|---|---|
| Email | 24/7 (מענה בשעות עבודה) | P3, P4 | 4-24 שעות |
| Chat (in-app) | 09:00-22:00 | P2, P3 | 30 דקות-4 שעות |
| Phone — תמיכה רגילה | 09:00-18:00 | P2 (Pro+), P3 | מיידי |
| Phone — חירום | 24/7 (Pro+) | P1 | 15 דקות |

## 3. איך לפתוח קריאת תמיכה טובה?

### מבנה מומלץ
1. **כותרת** — תיאור קצר וברור: "Cardcom נכשל בסליקה לכל הלקוחות"
2. **חומרה משוערת** — P1/P2/P3/P4
3. **מתי החלה הבעיה?**
4. **מי מושפע?** — כל הלקוחות / לקוח ספציפי / משתמש בודד
5. **מסך / Action שמשחזר**
6. **שגיאה מדויקת** (העתק + תמונה)
7. **מה ניסיתם** — Workarounds שניסיתם
8. **השפעה עסקית** — אם דחוף, פרט מה ההשפעה

### תבנית קריאה
```
Subject: [P1] [Cardcom] All payments failing since 14:30

Description:
- Started: 2026-05-17 14:30 IST
- Affected: All customers attempting checkout
- Error: "Cardcom API timeout"
- Tried: Restart browser, clear cache — no help
- Impact: Sales paused, ~30 orders queued
- Account: ACME-PROD
- Reproducer: Place any order, hit Pay button

Attachments: screenshot.png, console.log
```

## 4. תהליך טיפול

```
פתיחת קריאה
    ↓
שיוך אוטומטי לפי חומרה + נושא
    ↓
תגובה ראשונה (SLA)
    ↓
חקירה
    ↓
פתרון או Workaround
    ↓
אישור לקוח
    ↓
סגירה + Post-Mortem (P1/P2)
```
