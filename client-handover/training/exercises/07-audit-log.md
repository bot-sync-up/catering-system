# תרגיל 7 — קריאת Audit Log וזיהוי אירוע חשוד

## מטרה
להכיר את ה-Audit Log ולזהות פעולה חריגה.

## זמן
15 דקות

## הוראות

1. **Admin ← Audit Log**
2. סנן לפי 24 שעות אחרונות
3. סנן לפי `Severity: WARNING+`
4. עבור על כל הרשומות

### תרחיש לזיהוי
מתישהו במהלך התרגיל המדריך יבצע:
- 5 ניסיונות כניסה כושלים מאותו IP
- כניסה מוצלחת מ-IP בחו"ל
- מחיקת לקוח עם הזמנות פתוחות (פעולה רגישה)

## משימה
- זהה את 3 האירועים החשודים
- מלא טופס Incident Report:
  - תיאור האירוע
  - שעת התרחשות
  - IP מקור
  - משתמש מעורב
  - חומרה
  - פעולה מומלצת

## תוצרי משנה
- צילום מסך של ה-Audit Log המסונן
- Incident Report מלא

## קריטריון הצלחה
זיהוי 3 מתוך 3 + המלצה לפעולה (Force Logout / Reset Password / IP Block).
