# מצב אוטומטי מול מצב ידני (manual / auto / hybrid)

מערכת הדיווח של Catering תומכת בשלושה מצבי עבודה. ההחלטה ביניהם מוגדרת ב-ENV
`TAX_REPORTING_MODE`. רק "מנהל כללי" (general-manager) יכול לשנות אותה דרך הפורטל.

## auto - הגשה אוטומטית

**המערכת מייצרת ומגישה את כל הקבצים בעצמה.** ניתן רק אם:

- ללקוח יש כרטיס חכם דיגיטלי קבוע במערכת.
- חתום ייפוי כוח טכני שמאשר לנו להגיש.
- הלקוח מוותר על בקרה ידנית של הרו"ח.

**נדיר**. רוב הלקוחות לא יבחרו במצב הזה.

| יתרון | חסרון |
|-------|--------|
| 0 התערבות אנוש | סיכון משפטי גבוה אם יש טעות |
| מתאים לחברות גדולות עם CFO פנימי | קשה לבטל הגשה |

## manual - רואה חשבון מדווח (ברירת מחדל)

**המערכת מייצרת קבצים בלבד; הרו"ח מגיש ידנית.**

זהו ה-flow העיקרי של החבילה הזו. ראה [`README.md`](../README.md) לפרטים.

| יתרון | חסרון |
|-------|--------|
| הרו"ח שולט במועד ובערכים | תלוי בזמינות הרו"ח |
| הקטנת אחריות הספק (אנחנו) | רו"ח יכול לאחר ולגרום קנס |
| תואם לחוזים עם רוב משרדי רו"ח | דורש פורטל ייעודי |

## hybrid - חצי אוטומטי

**המערכת מגישה PCN874 אוטומטית, אך מחכה לאישור רו"ח לדוחות שנתיים ו-126.**

| הגשה אוטומטית | הגשה ידנית |
|--------------|------------|
| PCN874 חודשי | טופס 102 חודשי |
| (אופציונלי) טופס 102 | טופס 856 שנתי |
|  | טופס 126 שנתי |
|  | דוחות שנתיים |

מתאים ללקוחות מתקדמים שרוצים שקיפות חודשית אוטומטית אבל סקירה שנתית אנושית.

## איך משנים מצב

1. רק "מנהל כללי" של הלקוח רואה את ההגדרה.
2. כניסה ל-`/portal/settings/reporting-mode`.
3. בחירה ב-radio button + אישור.
4. כל שינוי מתועד ב-audit log עם userId, timestamp וערך קודם/חדש.

## איך בוחרים?

- **רוב הלקוחות** → `manual`.
- **לקוח עם CFO פנימי + כרטיס דיגיטלי** → `auto`.
- **לקוח שרוצה אוטומציה חודשית + סקירה שנתית** → `hybrid`.
