# מעבר ממצב auto למצב manual

מסמך מבצעי למעבר לקוח קיים ממצב **auto** למצב **manual** (רואה חשבון מדווח).

## למה לעבור?

- הרו"ח של הלקוח דרש שליטה.
- היו טעויות בהגשה אוטומטית שהובילו לדוחות מתקנים.
- שינוי במבנה הארגוני אצל הלקוח (CFO החדש רוצה לעבוד מול רו"ח חיצוני).
- דרישה משפטית מהביטוח של הלקוח.

## תהליך מסודר (4 שלבים)

### שלב 1 — תיאום עם הרו"ח של הלקוח

1. קבע פגישה (Zoom או פיזית).
2. הראה לו את ה[מדריך לרו"ח](./ACCOUNTANT-GUIDE.md).
3. הראה לו את ה[מדריך הגשה בשע"מ](./SHAAM-SUBMISSION-STEP-BY-STEP.md).
4. ודא שיש לו:
   - כרטיס חכם תקף
   - גישה לפורטל שע"מ
   - יכולת לקבל מייל + WhatsApp + SMS

### שלב 2 — יצירת משתמש פורטל

```bash
pnpm tsx scripts/create-accountant-user.ts \
  --email cpa@office.co.il \
  --name "רו\"ח דני כהן" \
  --client tax-id 514321987
```

המשתמש יקבל מייל הזמנה עם קישור להגדרת סיסמה + 2FA.

### שלב 3 — שינוי TAX_REPORTING_MODE

**חשוב:** את השינוי מבצע **המנהל הכללי של הלקוח**, לא אנחנו. זה מבטיח שהוא
מודע ומקבל אחריות.

1. המנהל נכנס ל-`/portal/settings/reporting-mode`.
2. בוחר **manual**.
3. מאשר במודאל "אני מבין שמעכשיו הרו"ח אחראי על ההגשה".
4. השינוי נכנס לתוקף **מהחודש הבא** (כדי לא לשבור הגשה שכבר תוזמנה).

### שלב 4 — מעבר נקי

- חודש המעבר: המערכת **מסיימת** את ההגשה האוטומטית האחרונה.
- מהחודש הבא: המערכת מייצרת קובץ ושולחת לרו"ח, מבלי להגיש.
- ב-15 לחודש הראשון אחרי המעבר — פגישת מעקב עם הרו"ח לוודא שהכל זרם.

## מה לא משתנה?

- מבנה הקבצים (XML/Mai101) — זהה לחלוטין.
- אלגוריתם החישוב — זהה.
- ארכיון הקבצים — נשמר כרגיל ל-7 שנים.
- audit log — ממשיך לתעד.

## מה כן משתנה?

| מצב auto | מצב manual |
|----------|-----------|
| `auto-submit-job` רץ ב-15 לחודש | רק `monthly-files-job` רץ ב-10 לחודש |
| אסמכתא נכנסת אוטומטית מ-API של שע"מ | אסמכתא מוקלדת ידנית ע"י הרו"ח |
| אם יש כישלון API → התראה למנהל | אם הרו"ח לא הגיש → התראה ל-WhatsApp+SMS |
| 0 התראות לרו"ח | 3 התראות (Email/WhatsApp/SMS) |

## Rollback (חזרה ל-auto)

לא מומלץ — אבל אפשרי:

1. המנהל הכללי שוב נכנס ל-`/portal/settings/reporting-mode`.
2. בוחר **auto** + מאשר חתימה דיגיטלית מחודשת על ייפוי הכוח.
3. השינוי נכנס לתוקף מהחודש הבא.

## בעיות אפשריות

| תרחיש | מה לעשות |
|--------|----------|
| הרו"ח לא קיבל הזמנה | בדוק SPAM, או צור מחדש דרך הסקריפט. |
| הרו"ח לא מצליח להוריד | בדוק הרשאות RBAC + שהקובץ סטטוס != pending. |
| הרו"ח טוען שהמספרים שונים מהציפיות | הראה לו את ה-audit log + checksum. צריך לאתר תזרים נתונים שגוי. |
| המעבר נעשה באמצע חודש | לעולם לא — תמיד מהראשון לחודש הבא. |
