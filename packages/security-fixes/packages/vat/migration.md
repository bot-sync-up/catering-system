# מיגרציה: שיעור מע"מ 17% → 18%

<div dir="rtl">

## רקע

החל מ-1 בינואר 2025 שיעור המע"מ בישראל עלה מ-17% ל-18%.
כל מערכת חיוב/חשבונית חייבת לחשב לפי שיעור התקף לתאריך החיוב.

## שלבי המיגרציה

### 1. עדכון קוד

החליפו כל מופע של:

```ts
const VAT = 0.17;
const total = price * 1.17;
```

ב:

```ts
import { getVatRate, withVat } from '@security-fixes/vat';
const total = withVat(price, invoiceDate);
```

### 2. סקריפט DB לעדכון רשומות 2025

> חשוב: עדכנו רק רשומות שתאריך החיוב שלהן ב-2025 ואילך והן עדיין בסטטוס `pending` או `draft`.
> אסור לעדכן חשבוניות שכבר נשלחו ללקוח / לרשות המסים.

```sql
-- PostgreSQL
BEGIN;

UPDATE invoices
SET    vat_rate    = 0.18,
       vat_amount  = ROUND(amount_excl_vat * 0.18, 2),
       total       = ROUND(amount_excl_vat * 1.18, 2),
       updated_at  = NOW()
WHERE  invoice_date >= '2025-01-01'
  AND  status IN ('draft', 'pending')
  AND  vat_rate = 0.17;

-- ודאו ספירה לפני commit
SELECT COUNT(*) FROM invoices WHERE vat_rate = 0.17 AND invoice_date >= '2025-01-01';
-- אם הספירה 0 → COMMIT, אחרת ROLLBACK ובדיקה

COMMIT;
```

### 3. עדכון תבניות / PDF / מיילים

חפשו בכל הפרויקט:

- מחרוזות "17%" → "18%"
- מחרוזות "מעמ 17" / "מע\"מ 17" → 18
- תבניות חשבונית (HTML/PDF/Handlebars) שמציגות שיעור מע"מ קשיח

### 4. בדיקת ספקי משלוח חשבוניות

ודאו ש-iCount / GreenInvoice / Rivhit מעודכנים לחישוב 18% עבור 2025.
חשבוניות עם תאריך 2024 עדיין יחושבו ב-17%.

### 5. דיווח לרשות המסים

- טופס 102 (דיווח חודשי) — עמודות מע"מ עסקאות ומע"מ תשומות חייבות לשקף 18%.
- טופס 126 — אין שינוי בשיעור (זה מס הכנסה).

## Rollback

במקרה הצורך: `process.env.VAT_RATE_OVERRIDE=0.17` יחזיר את כל המערכת ל-17%
זמנית, עד שיוחלט מה לעשות עם רשומות שכבר חושבו ב-18%.
