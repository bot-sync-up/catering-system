# מסמך ציות - דרישות רשות המסים בישראל

## אישור תוכנה
- **מספר אישור:** 1346
- **שם רשמי:** iCount
- **תוקף:** בהתאם להחלטות הרשות

## מספרי הקצאה - מודל ישראל

### החלת מודל הדיווח המקוון
החל מינואר 2024, חשבונית מס שסכומה (לפני מע"מ) עולה על סף מסוים מחייבת קבלת **מספר הקצאה** משירותי שע"מ. ללא מספר הקצאה, החשבונית אינה מהווה תיעוד פנים מותר בניכוי תשומות.

### ספים לשנים הקרובות
| שנת מס | סף בש"ח (לפני מע"מ) |
|--------|---------------------|
| 2024 | 25,000 |
| 2025 | 20,000 |
| 2026 | 10,000 |
| 2027 ואילך | 5,000 |

### זרימת הבקשה
1. הלקוח מבקש להפיק חשבונית מס
2. הספרייה מזהה אם נדרש מספר הקצאה
3. שליחת בקשה ל-iCount (`/allocation_number/get`)
4. iCount פונה לשירותי שע"מ, מקבל מספר הקצאה ומחתים אותו דיגיטלית
5. הספרייה מטמיעה את מספר ההקצאה במסמך הסופי
6. במקרה כשל - שגיאה ברורה ללקוח

## דוח PCN874 (מבנה אחיד)
פורמט הדיווח החדש של רשות המסים. מסמך טקסט עם רשומות הפרדה:
- **A** - רשומת פתיחה (תאריכים, סיכומים)
- **B** - רשומות פירוט לכל מסמך
- **Z** - רשומת סיכום

הספרייה מפיקה את הקובץ דרך `vatReportService.formatAsPCN874(report)`.

## תיעוד וביקורת
- כל פעולת API נרשמת ב-`IntegrationLogs`
- שמירה למשך 7 שנים (דרישת הרשות)
- שדות חובה: timestamp, operation, success, documentId, allocationNumber, amount

## אבטחה
- API Key מוצפן בעת מנוחה
- Webhook signature באמצעות HMAC-SHA256
- Retry בעדינות (exponential backoff) למניעת עומס על שע"מ
- אין לוג של מספרי כרטיסי אשראי או נתוני אבטחה
