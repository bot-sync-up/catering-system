# הצהרת תאימות — תוכנה מאושרת מס' 1346

## רקע משפטי

חבילה זו מתאימה לדרישות הבאות:

1. **הוראות מס הכנסה (ניהול פנקסי חשבונות), התשל"ג-1973**
2. **תקנות מס ערך מוסף (ניהול פנקסי חשבונות), התשל"ו-1976**
3. **חוק לצמצום השימוש במזומן, התשע"ח-2018**
4. **תיקון חוק העוסקים — מודל ישראל (Israel Model)**

מספר התוכנה המאושרת: **1346**

## מודל ישראל — ספי הקצאה

החל מ-1 בינואר 2024, חשבוניות מס מעל לסף מסוים מחייבות
**מספר הקצאה** (Allocation Number) מרשות המסים לפני הנפקתן.

| שנה | סף ב-₪ (כולל מע"מ) |
| ---- | -------------------: |
| 2024 |              25,000 |
| 2025 |              20,000 |
| 2026 |              10,000 |
| 2027 |               5,000 |

**סוגי מסמכים החייבים בהקצאה:**
- חשבונית מס
- חשבונית מס/קבלה
- חשבונית מס זיכוי
- החזרה (סוגי refund)

**מסמכים פטורים:**
- חשבונית עסקה (לא מס)
- קבלה בלבד
- הצעת מחיר
- הזמנה
- תעודת משלוח

## דרישות התוכנה המאושרת

### תיעוד וחתימה דיגיטלית

כל חשבונית חייבת לכלול:
1. מספר תוכנה מאושרת (1346)
2. שם החברה המפעילה
3. גרסת התוכנה
4. חתימה דיגיטלית (hash)

הפונקציה `buildSoftware1346Declaration()` מייצרת הצהרה
מלאה עם hash SHA-256 על השדות הקבועים.

### Footer חובה לחשבונית מודפסת

עברית: `תוכנה מאושרת מס' 1346 - רשות המסים בישראל`
אנגלית: `Approved Software No. 1346 - Israeli Tax Authority`

## דוחות חובה

### PCN874 / Mai101 — דוח דו-חודשי למע"מ

- **רשומה A** — מכירות (חשבוניות שיצאו)
- **רשומה B** — קניות (חשבוניות שהתקבלו)
- **רשומה Z** — סיכום

פורמט: טקסט קבוע-רוחב, CRLF, ASCII (UTF-8 בגרסה החדשה).

### טופס 126 — שנתי, ניכויי שכר

דוח שנתי המוגש על-ידי מעבידים, ומפרט לכל עובד:
- ת.ז.
- שכר ברוטו
- מס הכנסה
- ביטוח לאומי
- מס בריאות

### טופס 856 — שנתי, לקוחות וספקים

דוח שנתי המפרט לכל לקוח/ספק:
- ע.מ. / ח.פ.
- מספר תנועות
- סה"כ ללא מע"מ
- סה"כ מע"מ

## שמירה ל-7 שנים

לפי פקודת מס הכנסה, סעיף 130, חובה לשמור:
- ספרי חשבונות
- חשבוניות, קבלות, תעודות משלוח
- חוזים ותעודות נלוות

למשך **7 שנים מתום שנת המס** שבה הופקו.

החבילה מארכבת אוטומטית ל-Cloudflare R2 / S3
תחת **Object Lock במצב COMPLIANCE**, כך שלא ניתן למחוק אותם
עד לתום תקופת ההחזקה.

כל קובץ נשמר עם:
- SHA-256 hash במטא-דאטה
- תאריך פקיעת ההחזקה
- מזהי המסמך, השנה, וה-CID

## אבטחה ואותנטיות

### חתימת Webhooks

iCount חותם webhooks ב-HMAC-SHA256 עם secret משותף.
המקלט שלנו מבצע:
- **אימות חתימה** ב-`constant-time compare`
- **Replay protection** — דחיית timestamps ישנים (default 5 דקות)
- **Schema validation** — Zod

### Logging מלא

כל קריאת API נשמרת ב-`IntegrationLog` עם:
- request payload (מסונכן — passwords/tokens מוסתרים)
- response payload
- status code
- duration
- מספר ניסיון
- error (אם יש)

## רשימת בדיקות יישום
